    • The Yin and Yang of Software Quality: Do Design Patterns Induce Code Smells?

    Giammaria Giordano, Giulia Sellitto, Aurelio Sepe, Fabio Palomba, Filomena Ferrucci

    giagiordano@unisa.it, gisellitto@unisa.it, a.sepe21@studenti.unisa.it, fpalomba@unisa.it, fferrucci@unisa.it

    University of Salerno (Italy)

    Abstract

    Software reuse is considered the silver bullet of software engineering. It has been largely demonstrated that the proper implementation of design and reuse principles can substantially reduce the effort, time, and costs required to develop software systems. Among the available techniques for source code reuse, one of the most affirmed is the use of design patterns i.e., the reuse of existing solutions to deal with a similar problem when implementing new features or refactoring the existing ones. While previous work pointed out the benefits that design patterns provide in terms of maintainability and understandability, some studies seem to raise the opposite concern, suggesting that they can negatively impact the code quality from the developersâ€™ perspectives. We recognize such discrepancy in the literature and we aim to fill this gap by investigating whether and how design patterns can induce the emergence of issues compromising code understandability, namely the Complex Class, God Class, and Spaghetti Code smells, which have been also shown to increase the change- and fault-proneness of code. We perform an empirical evaluation on 15 JAVA projects evolving over 542 releases, and we find that, although design patterns are supposed to improve code quality without prejudice, they can lead to dangerous issues by inducing the emergence of code smells in the classes participating in their implementation.
